# HTTP Tunnels

An HTTP tunnel forwards public HTTPS traffic at `https://<subdomain>.binboi.com` to a local HTTP port on your machine.

## How it works

```
Browser ──HTTPS──▶ https://my-app.binboi.com ──tunnel──▶ http://localhost:3000
```

1. The CLI opens a long-lived connection to the Binboi edge.
2. The edge claims `<subdomain>.binboi.com` for that connection.
3. Incoming HTTPS requests are decrypted at the edge and forwarded through the tunnel to your local port.
4. Responses flow back through the same connection.

Your local server only speaks plain HTTP — TLS terminates at the edge.

## Open a tunnel

```bash
binboi http 3000 my-app
```

This requires:

- An authenticated CLI (`binboi login`).
- An available subdomain (one not currently used by another account).
- Remaining capacity under your plan's tunnel and request quotas.

## Quotas

Every HTTP request through a tunnel counts against your daily request budget.

| Plan | Active tunnels | Requests / day |
|---|---|---|
| Free | 1 | 100 |
| Pro | 10 | 10,000 |
| Max | 50 | Unlimited |

When the daily limit is hit, the edge returns `429 Too Many Requests` until the next UTC day rolls over.

## Close a tunnel

Press `Ctrl+C` in the terminal that started it. The subdomain is released back to the pool immediately.

## Custom domains

Pro and Max plans can attach custom domains (10 and 50 respectively) at [/dashboard](https://binboi.com/dashboard). Free accounts are limited to `*.binboi.com`.
