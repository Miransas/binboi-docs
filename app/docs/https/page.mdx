# HTTPS & TLS

Every Binboi tunnel terminates TLS at the edge with a valid certificate.

## How It Works

When you run `binboi http 3000`, the public URL `https://abc123.binboi.com` serves a wildcard certificate issued for `*.binboi.com`. Your local service can run plain HTTP — Binboi handles the TLS handshake on the public side.

## End-to-End TLS

If your local service speaks HTTPS, Binboi can pass the TLS connection through unmodified:

```bash
binboi http 3000 --tls
```

This is useful when your local service has its own certificate (e.g. for mTLS testing).

## Custom Certificates (Pro+)

For custom hostnames, Binboi automatically requests a Let's Encrypt certificate. You can also upload your own:

```bash
binboi http 3000 --hostname tunnel.example.com \
  --cert /path/to/cert.pem \
  --key /path/to/key.pem
```

## TLS Version

Binboi enforces TLS 1.2+ on the public side. Older clients receive a `protocol_version` alert.
