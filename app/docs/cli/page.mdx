# CLI Reference

The `binboi` binary is the primary way to open tunnels and inspect your account from the terminal.

## Commands

### `binboi login --token <PAT>`

Saves a PAT to `~/.binboi/config.json`. All other commands read it from there.

```bash
binboi login --token binboi_pat_xxxxxxxx_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### `binboi whoami`

Prints the account email and current plan for the saved PAT.

```bash
binboi whoami
```

### `binboi http <port> <subdomain>`

Opens an HTTP tunnel from `https://<subdomain>.binboi.com` to `http://localhost:<port>`.

```bash
binboi http 3000 my-app
# → https://my-app.binboi.com
```

The subdomain must be available and within your plan's tunnel quota. Press `Ctrl+C` to close the tunnel.

### `binboi tunnels`

Lists your active tunnels with their subdomain, target port, and status.

```bash
binboi tunnels
```

### `binboi --help`

Shows the full command list and flags. Each subcommand also accepts `--help`.

```bash
binboi --help
binboi http --help
```

## Environment variables

| Variable | Purpose | Default |
|---|---|---|
| `BINBOI_API_URL` | Control plane base URL | `https://api.binboi.com` |
| `BINBOI_AUTH_TOKEN` | PAT used instead of the saved config | — |
| `BINBOI_SERVER_ADDR` | Tunnel server address | production default |

Setting `BINBOI_AUTH_TOKEN` is useful in CI — you can skip `binboi login` and pass the token through the environment.

## Configuration file

After `binboi login`, the CLI writes:

```
~/.binboi/config.json
```

It stores the active PAT and any cached account info. Delete the file to fully sign out.
