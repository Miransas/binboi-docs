# HTTP Tunnels

Expose any local HTTP service to a public URL with automatic TLS.

## Basic Usage

```bash
binboi http 3000
```

Output:

```
Tunnel started
  Public URL : https://abc123.binboi.com
  Forwarding : https://abc123.binboi.com → http://localhost:3000
  Region     : eu-fra
```

Open the URL in any browser. Requests are forwarded to your local port over a multiplexed TCP connection — latency is near real-time.

## Reserved Subdomains (Pro+)

Keep the URL stable across sessions:

```bash
binboi http 3000 --subdomain myapp
# https://myapp.binboi.com
```

## Custom Hostnames (Pro+)

Use your own domain. First, add a CNAME record:

```
tunnel.example.com  CNAME  cname.binboi.com.
```

Then:

```bash
binboi http 3000 --hostname tunnel.example.com
```

Binboi provisions a TLS certificate automatically via Let's Encrypt.

## Basic Auth

Protect the public URL with HTTP basic auth:

```bash
binboi http 3000 --basic-auth alice:secret
```

Browsers prompt for the credentials before any request reaches your local server.

## Inspecting Requests

Run with `--inspect` to open a local dashboard at `http://localhost:4040`:

```bash
binboi http 3000 --inspect
```

You'll see every request, response, header, and body — useful for webhook debugging.
