# API Gateway

Binboi can act as a public API gateway in front of your local services — useful for splitting traffic across multiple ports from a single hostname.

## Coming Soon

This feature is on the roadmap. Today, run one tunnel per service:

```bash
binboi http 3000 --subdomain api
binboi http 8080 --subdomain admin
```

Path-based routing rules (e.g. `/api/*` → port 3000, `/admin/*` → port 8080) are planned for a future release. Track progress on [GitHub](https://github.com/Miransas/binboi).

## Workaround

Use a local reverse proxy (Caddy, nginx) that fronts your services, then expose the proxy:

```bash
# Caddyfile
:8000 {
    handle /api/* { reverse_proxy localhost:3000 }
    handle /admin/* { reverse_proxy localhost:8080 }
}
```

Then:

```bash
binboi http 8000 --subdomain myapp
```
